# รีวิวกลยุทธ์ Quorum — แข็งแรงแค่ไหน + เพิ่มอัตราชนะยังไง

> สรุปจากการตรวจโค้ดจริง (agents / aggregator / judge / risk) — อัปเดต 2026-06

## ⚠️ ความจริงที่ต้องเข้าใจก่อน: "อัตราชนะ" ไม่เท่ากับ "กำไร"

นี่สำคัญมาก — **อย่าไล่ตามอัตราชนะอย่างเดียว** มันเป็นกับดัก:
- ชนะ 90% แต่ไม้แพ้ขาดทุนหนัก → **ขาดทุนรวม** ได้
- ชนะ 40% แต่ไม้ชนะกำไรเยอะ ไม้แพ้ขาดทุนน้อย → **กำไรรวม**

ตัวเลขที่ควรดูคือ **expectancy** = (อัตราชนะ × กำไรเฉลี่ย) − (อัตราแพ้ × ขาดทุนเฉลี่ย)
และ **profit factor** (มีใน Dashboard แล้ว) > 1 = ระบบมีกำไร

ความสัมพันธ์กับ TP/SL: ถ้าตั้ง `take_profit=5%`, `stop_loss=3%` → ต้องชนะ > **37.5%** ถึงเท่าทุน
อยากได้ "อัตราชนะสูง" → ตั้ง TP เล็ก SL ใหญ่ (แต่เสี่ยงไม้แพ้หนัก) — ต้อง **สมดุล** ไม่ใช่สุดโต่ง

## ✅ จุดแข็งของกลยุทธ์ปัจจุบัน

1. **Ensemble ไม่เชื่อ AI ตัวเดียว** — 5 ที่ปรึกษาโหวต ลดความผิดพลาดของโมเดลเดี่ยว
2. **มองหลายมุม** — technical + momentum + sentiment การเงิน (FinBERT) + sentiment คริปโต (CryptoBERT) + ข่าว
3. **Judge มี web search** — ใช้ข้อมูลสดจากเน็ต ไม่พึ่งความรู้เก่า
4. **Veto ข่าวร้ายแรง** — hack/ban/delist หยุดเทรดทันที
5. **Risk gate** — min_confidence + จำกัดจำนวน position

## ⚠️ จุดอ่อน + สิ่งที่แก้แล้วรอบนี้

| จุดอ่อน | สถานะ |
|---------|-------|
| **ไม่มีระบบ "ปิดไม้"** → วัดแพ้/ชนะไม่ได้เลย | ✅ **แก้แล้ว** — auto take-profit/stop-loss ปิดไม้อัตโนมัติ |
| **เทรดบนข้อมูลปลอม** (ตอน offline ใช้ราคา synthetic) | ✅ **แก้แล้ว** — ไม่เทรดถ้า data เป็น synthetic |
| **สั่งขายของที่ไม่มี** → failed รัวๆ | ✅ **แก้แล้ว** — ข้ามคำสั่งขายถ้าไม่มี position |
| **technical + trend_ml ซ้ำกัน** (momentum ทั้งคู่) | ⏳ ควรเพิ่ม agent ที่มองตรงข้าม |
| **sentiment agent ต้องมีข่าว** ถ้าไม่มีข่าว = เหลือผู้โหวต 2-3 ตัว | ⏳ ปรับ threshold แบบ dynamic |
| **ไม่มี regime filter** (เทรด trend ในตลาด sideway = เจ๊ง) | ⏳ |
| **ไม่มี backtest** — พิสูจน์ edge ก่อนเสี่ยงเงินไม่ได้ | ⏳ **สำคัญสุด** |

## 🎯 แผนเพิ่มอัตราชนะ/expectancy (เรียงตามผลกระทบ)

### 1. ⭐ Backtesting harness (สำคัญที่สุด)
"เพิ่มอัตราชนะ" ทำไม่ได้ถ้า **วัดออฟไลน์ไม่ได้** — ต้องเอา candle ย้อนหลังมารันผ่าน pipeline
แล้ววัด win rate / expectancy / max drawdown ของแต่ละชุดพารามิเตอร์ ก่อนใช้เงินจริง
→ ปรับ weights, threshold, TP/SL บนข้อมูลจริงได้โดยไม่เสียเงิน

### 2. ⭐ ปิดไม้ให้ฉลาดขึ้น (ต่อยอดจาก TP/SL ที่เพิ่งทำ)
- **Trailing stop** — ล็อกกำไรเมื่อราคาขึ้นแล้วย่อ
- **Time stop** — ถ้าถือเกิน N ชั่วโมงแล้วไม่ไปไหน ปิด
- ปรับ TP/SL ตาม volatility (ATR) แทนค่า % คงที่

### 3. เพิ่มความหลากหลายของที่ปรึกษา (ลดสัญญาณซ้ำ)
- เพิ่ม **mean-reversion agent** (RSI สุดขั้ว / Bollinger) — มองตรงข้ามกับ 2 momentum agent
- เพิ่ม **volume/orderbook agent** — ดูแรงซื้อขายจริง ไม่ใช่แค่ราคา
- เอเจนต์ที่ไม่ correlate กัน = การโหวตมีความหมายจริง

### 4. Regime filter — เลือกเวลาเทรด
- คำนวณ ADX/volatility → **เทรดเฉพาะตอนตลาดมีเทรนด์ชัด** นั่งเฉยตอน sideway
- ลดจำนวนไม้เสีย (ส่วนใหญ่ขาดทุนตอนตลาดไม่มีทิศทาง)

### 5. Liquidity filter + cooldown
- เทรดเฉพาะเหรียญ volume สูง (เลี่ยงปั่น/slippage)
- ห้ามเข้าเหรียญเดิมซ้ำใน N นาที (กัน overtrading + ค่าธรรมเนียม)

### 6. ปรับน้ำหนัก agent ตามผลงานจริง (มีข้อมูลใน DB แล้ว)
- เก็บสถิติว่า agent ไหนทำนายถูกบ่อย → เพิ่มน้ำหนัก, ตัวที่มั่วลดน้ำหนัก
- ปรับ `min_confidence` ให้สูงขึ้นในโหมด live (เลือกเฉพาะไม้คุณภาพ)

### 7. Position sizing ตามความมั่นใจ
- มั่นใจมาก = ลงเยอะขึ้น (แบบ Kelly-lite) แทนเงินคงที่ทุกไม้

## สรุปสั้น

กลยุทธ์ปัจจุบัน **มีโครงที่ดี** (ensemble + judge + web search) แต่ก่อนหน้านี้ **ยังไม่ครบวงจร**
เพราะไม่มี "ทางออก" — รอบนี้เติม **TP/SL auto-exit** ทำให้วัดแพ้/ชนะได้จริงแล้ว

ขั้นต่อไปที่จะดันอัตราชนะ/กำไรได้จริงคือ **backtesting** (วัด+จูนออฟไลน์) และ
**เพิ่ม agent ที่หลากหลาย + regime filter** (ลดไม้เสีย) — โฟกัส **expectancy ไม่ใช่ win rate ล้วน**
